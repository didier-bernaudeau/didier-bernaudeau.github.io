Edge Router on Dedibox

Date dim. 03 septembre 2017 Modified mar. 16 janvier 2018 Tags Network / ArchLinux / Dedibox

An edge router ensures the connectivity of its network with external networks. In my case, this router is a Dedibox hosted by Online at Amsterdam and ensures the connectivity between my LAN and Internet through a VPN network. This post describes the first step to configure ArchLinux.

Operating System

My Edge Router run with ArchLinux.

  • Change password for main user
$ passwd
  • Change password for root user
$ sudo su
$ passwd
  • Sync repository
$ sudo pacman -Sy
  • Update system
$ sudo pacman -Su
  • Configure nano (text editor)
$ sudo nano /etc/nanorc

set nowrap
include "/usr/share/nano/*.nanorc"

Fail2ban

  • Configure SSH daemon
$ sudo nano /etc/ssh/sshd_config

LogLevel VERBOSE
  • Restart SSH daemon
$ sudo systemctl restart sshd
  • Install fail2ban
$ sudo pacman -S --noconfirm fail2ban
  • Configure fail2ban for SSH daemon
$ sudo nano /etc/fail2ban/jail.d/jail.conf

[DEFAULT]
bantime = 864000
ignoreip = 127.0.0.1/8 10.0.0.0/24 10.XXX.3.0/24


[sshd]
enabled  = true
filter   = sshd
action   = iptables[name=SSH, port=ssh, protocol=tcp]
backend  = systemd
maxretry = 5
  • Start fail2ban
$ sudo systemctl start fail2ban.service
$ sudo systemctl enable fail2ban.service
  • Read log (for debugging purposes)
$ sudo journalctl -b -u fail2ban -n 50

ARP Wathching

Arpwatch is an open source computer software program that helps you to monitor Ethernet traffic activity (like Changing IP and MAC Addresses) on your network and maintains a database of ethernet/ip address pairings.

  • Install ARP Watch (in progress)
$ sudo pacman -S --noconfirm arpwatch

Select msmtp-mta as smtp-forwarder

  • Create unit file for SystemD
$ sudo nano /etc/systemd/system/arpwatch.service

[Unit]
Description=Arpwatch daemon which keeps track of ethernet/ip address pairings
After=network.target

[Service]
Type=forking
ExecStart=/usr/sbin/arpwatch -u arpwatch -e root -s 'root (Arpwatch)'

[Install]
WantedBy=multi-user.target
  • Start ARP Watch
$ sudo systemctl start arpwatch.service
$ sudo systemctl enable arpwatch.service
