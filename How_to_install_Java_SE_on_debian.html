How to install Java SE on Debian

Date jeu. 12 janvier 2017 Modified ven. 13 janvier 2017 Tags Debian / Java

Lately, install Java SE on Linux has become a nightmare because most of Linux distributions, such as Debian, have chosen to remove this package from official repositories due to license reasons. However, Java SE is needed for some applications which are not compatible with Open JRE / JDK. So, this post describes step by step how to install Java SE (JRE and JDK) provided by Oracle on Debian.

Java Package

Java-package is a tool that provides the ability to build a Debian package from a Java binary.

  • Add a "contrib" component in sources.list
# nano /etc/apt/sources.list

deb http://*********.debian.org/debian/ jessie main contrib
  • Update the list of available packages
# apt-get update
  • Install the java-package package
# apt-get install java-package
  • Install packages needed by Java 8
# apt-get install libxslt1-dev

Java Runtime Environment

JRE (Java Runtime Environment) contains the JVM (Java Virtual Machine) to execute java programs and other java tools such as keytool.

  • Download Java 8 Standard Edition Runtime Environment
$ wget --header "Cookie: oraclelicense=accept-securebackup-cookie" http://download.oracle.com/otn-pub/java/jdk/8u112-b15/jre-8u112-linux-x64.tar.gz
  • Create a Debian package
$ fakeroot make-jpkg jre-8u112-linux-x64.tar.gz
  • Install the binary package
# dpkg -i oracle-java8-jre_8u112_amd64.deb

Java Development Kit

The JDK (Java Development Kit) includes JRE and tools for developers to create and compile java programs (javac, JavaDoc, Java Debugger, ...).

  • Download Java 8 Standard Edition Development Kit
$ wget --header "Cookie: oraclelicense=accept-securebackup-cookie" http://download.oracle.com/otn-pub/java/jdk/8u112-b15/jdk-8u112-linux-x64.tar.gz
  • Create a Debian package
$ fakeroot make-jpkg jdk-8u112-linux-x64.tar.gz
  • Install the binary package created:
# dpkg -i oracle-java8-jdk_8u112_amd64.deb

Select Java SE

  • Select the JVM of your choice
# update-alternatives --config java

There are 3 choices for the alternative java (providing /usr/bin/java).

  Selection    Path                                            Priority   Status
------------------------------------------------------------
  0            /usr/lib/jvm/java-7-openjdk-amd64/jre/bin/java   1071      auto mode
  1            /usr/lib/jvm/java-7-openjdk-amd64/jre/bin/java   1071      manual mode
* 2            /usr/lib/jvm/jdk-8-oracle-x64/jre/bin/java       318       manual mode
  3            /usr/lib/jvm/jre-8-oracle-x64/bin/java           317       manual mode

Press enter to keep the current choice[*], or type selection number: 3
  • Verify the selected JVM
# java -version

java version "1.8.0_112"
Java(TM) SE Runtime Environment (build 1.8.0_112-b15)
Java HotSpot(TM) 64-Bit Server VM (build 25.112-b15, mixed mode)
  • Select the Java compiler of your choice
update-alternatives --config javac

There are 2 choices for the alternative javac (providing /usr/bin/javac).

  Selection    Path                                         Priority   Status
------------------------------------------------------------
  0            /usr/lib/jvm/java-7-openjdk-amd64/bin/javac   1071      auto mode
  1            /usr/lib/jvm/java-7-openjdk-amd64/bin/javac   1071      manual mode
* 2            /usr/lib/jvm/jdk-8-oracle-x64/bin/javac       318       manual mode

Press enter to keep the current choice[*], or type selection number: 2
  • Verify the selected Java compiler
# javac -version

javac 1.8.0_112
