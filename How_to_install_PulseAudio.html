How to install PulseAudio on Gentoo Linux

Date lun. 19 octobre 2015 Modified dim. 21 janvier 2018 Tags Gentoo

A sound server is software that manages the use of and access to audio devices. There are many sound server (JACK Audio Connection Kit, Enlightened Sound Daemon, ...) but PulseAudio is now an integral part of all relevant modern Linux distributions. This tutorials describes how to install PulseAudio on Gentoo Linux.

PulseAudio Architecture

Install PulseAudio

  • Define local USE flags
# nano /etc/portage/package.use

media-sound/pulseaudio          -tcpd -webrtc-aec
  • Install PulseAudio
# emerge --oneshot pulseaudio

Configure capture device

  • List all capture devices
# arecord -l

**** List of CAPTURE Hardware Devices ****
card 1: PCH [HDA Intel PCH], device 0: ALC3232 Analog [ALC3232 Analog]
  Subdevices: 1/1
  Subdevice #0: subdevice #0

The card number is "1" and device number is "0". So, In hw:CARD,DEVICE notation, the capture device is hw:1,0.

  • Define the capture device to load audio drivers statically
# nano /etc/pulse/default.pa

load-module module-alsa-source device=hw:1,0
  • Restart pulse (by non-root user)
$ pulseaudio -k ; pulseaudio -D

Compile application to support PulseAudio

  • Define global USE flags
# nano etc/portage/make.conf

USE="pulseaudio"
  • Install packages
# emerge ffmpeg vlc kmix chromium
