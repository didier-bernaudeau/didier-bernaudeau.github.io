EnOcean

Date mer. 05 juillet 2017 Modified mar. 11 juillet 2017 Tags Home automation / Network / Wireless

EnOcean is an energy harvesting wireless technology used primarily in building automation systems and smart homes. All modules based on this technology combine on the one hand micro energy converters with ultra low power electronics, and on the other hand enable wireless communications between battery-less wireless sensors, actors and even gateways.

Protocols

ESP (Enocean Serial Protocol)

This protocol is designed to transfer information between EnOcean device and host using UART interface. For more information, read the specification v1.30 / March 22, 2016.

ERP (Enocean Radio Protocol)

This protocol is designed to sent information, as "EnOcean telegram", between two devices. According to the OSI layer model, this protocol covers layers 1 to 3. For more information, read the specification v2 / September 26, 2013.

EEP (EnOcean Equipment Profiles)

This specification describes the payload of "EnOcean telegram" for each profile.

The name of an equipment profile is divided in 3 elements:

  • RORG: the ERP radio telegram type (range: 00…FF, 8 Bit)
  • FUNC: basic functionality of the data content (range 00…3F, 6 Bit)
  • TYPE: type of device in its individual characteristics (range 00…7F, 7 Bit)

For example, the profile "A5-02-05":

  • A5: 4BS communication
  • 02: Temperature Sensors
  • 05: Temperature Sensors Range 0°C to +40°C

For more information, read the specification v2.6.3 / June 8, 2015.

Security

Wireless communication raises two significant concerns:

  • Unauthorized interception (reception and correct interpretation) of transmitted data (Eavesdropping)
  • Unauthorized transmission of correct control commands including “Replay Attacks"

Content protection addressing replay attacks:

  • Rolling Code: code that changes according to a predefined algorithm
  • CMAC: Cipher-based message authentication code with dynamic key (derived from rolling code)

Content protection addressing eavesdropping:

  • AES 128 bits CBC mode
  • Variable AES 2128 bits (data is encrypted using the 128-bit AES algorithm in combination with the rolling code)

The manufacturer is free to choose whether or not to implement these security mechanisms. Take care when selecting EnOcean modules!

For more information, read the Security of EnOcean Radio Networks v1.9 / July 26, 2013.
