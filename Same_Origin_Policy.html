SOP (Same-Origin Policy)

Date jeu. 27 août 2015 Modified jeu. 27 août 2015 Tags Development / WebSite / HTML / JavaScript / PHP

SOP (Same Origin Policy) is a security mechanism implemented in browsers to restrict interactions between 2 different origin resources. Two resources have the same origin if the protocol, port, and host are the same for both resources.

This article shows how to test the SOP (Same Origin Policy) mechanism implemented by your browser.

Step 1 : Create WebService

Create a PHP script named "sop.php" and hosted on "www.example.com" :

<?
  echo date("Y-m-d H:i:s");
?>

Step 2 : Create HTML web page

Create a HTML file named "sop.html" and hosted on your localhost :

<!doctype html>
<html>
 <body>

 <script>
  var xmlHttp = new XMLHttpRequest();
  xmlHttp.open( "GET", "http://www.example.com/sop.php", false );
  xmlHttp.send( null );
  alert(xmlHttp.responseText);
 </script>

 </body>
</html>

Step 3 : Display error message

Open the file "sop.html" in a browser. The cross-origin request is blocked by the browser and the following message is displayed in the debbugging console :

Cross-Origin Request Blocked: The Same Origin Policy disallows reading the remote resource at http://www.example.com/sop.php. This can be fixed by moving the resource to the same domain or enabling CORS.
