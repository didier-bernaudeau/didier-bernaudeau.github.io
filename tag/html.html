CORS (Cross-Origin Resource Sharing)

Date dim. 30 août 2015 Modified mar. 01 septembre 2015 Tags Development / WebSite / HTML / JavaScript / PHP

CORS (Cross-Origin Resource Sharing) is a mechanism, define by W3C, which provides a way for web servers to support cross-site access controls, which enable secure cross-site data transfers.

JSONP (JSON with Padding)

Date sam. 29 août 2015 Modified sam. 29 août 2015 Tags Development / WebSite / HTML / JavaScript / PHP

This article shows how JSONP (JavaScript Object Notation with Padding) can be used to get data from different domains, bypassing SOP (Same-Origin Policy).

JSONP uses the script tag, which is not in the scope of the SOP. Indeed, a JavaScript source file loaded from another domain will run in the origin of the HTML that includes the script, not in the domain where the JavaScript file originated from.

SOP (Same-Origin Policy)

Date jeu. 27 août 2015 Modified jeu. 27 août 2015 Tags Development / WebSite / HTML / JavaScript / PHP

SOP (Same Origin Policy) is a security mechanism implemented in browsers to restrict interactions between 2 different origin resources. Two resources have the same origin if the protocol, port, and host are the same for both resources.
