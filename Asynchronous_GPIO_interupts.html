Asynchronous GPIO interupts

Date dim. 15 avril 2018 Modified sam. 14 avril 2018 Tags Python / Rpi.GPIO / Raspberry Pi

Rpi.GPIO is a powerful framework to control GPIO on Raspberry. However, this framework is not really compatible with asynchronous programming.

Actually, a part of the documentation says: "the callback functions are run sequentially, not concurrently. This is because there is only one thread used for callbacks, in which every callback is run, in the order in which they have been defined"

So, this post describes a workaround to make asynchronous GPIO interupts.

For this example, a switch button is connected on 3,3V and GPIO 4 (PIN 7).

import asyncio
import RPi.GPIO as GPIO
import sys

loop = None

def FunctionEventCallback(Channel):
    print("Event detected: "+str(Channel))
    if GPIO.input(7):
        print("closed")
    else:
        print("Open")

def StartEventThread(PinGPIO=0):
    print("Pin: "+str(PinGPIO))
    loop.call_soon_threadsafe(FunctionEventCallback, PinGPIO)

if __name__ == '__main__':
    try:
        # setup the GPIO
        GPIO.setwarnings(True)
        GPIO.setmode(GPIO.BOARD)
        GPIO.setup(7, GPIO.IN, pull_up_down=GPIO.PUD_DOWN)
        GPIO.add_event_detect(7, GPIO.BOTH, callback=StartEventThread, bouncetime=300)

        # run the event loop
        loop = asyncio.get_event_loop()
        loop.run_forever()
        loop.close()
    except :
        print("Error:", sys.exc_info()[0])

    # cleanup
    GPIO.cleanup()

Then run it :

$ python3.5 script.py

Reminder: the framework "rpi-gpio.js" supports asynchronous programming but it doesn't support pull up/down resistor yet. Wait & see !
