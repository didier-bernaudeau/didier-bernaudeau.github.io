TLS : Understand the cipher suites

Date mer. 22 juillet 2015 Modified mer. 22 juillet 2015 Tags Cryptography / Network

To protect your communications, the TLS protocol uses 4 types of cryptographic algorithm : key exchange, authentication, encryption and message authentication code. The cipher suite allows to define the algorithm used for each of these 4 functions.

Types of cryptographic algorithm

Key exchange

This type of algorithm is used to define a common secret between two parties. The available algorithms are RSA, DH, DHE and ECDHE.

Authentication

This type of algorithm is used to authenticate the server by client and optionally client by server. The available algorithms are RSA or DSS.

Encryption

This type of algorithm is used to encrypt data. The available algorithms are DES, 3DES, RC4, AES, ...

Message Authentication Code

This type of algorithm is used to sign data. The available algorithms are MD5, SHA, SHA256, ...

Cipher suites

An cipher suite is a list of one or more cipher strings separated by a special char (colons, commas, spaces, ...). The name of the cipher suites is formalized by IANA (Internet Assigned Numbers Authority).

The cipher suite defines the algorithms used for key exchange, authentication, encryption then message authentication code. If the algorithm for key exchange is not mentioned, the default algorithm is RSA.

The table below details some example of cipher suites :

Value Cipher Suite Key Exchange Authentication Encryption MAC
0x00,0x04 TLS_RSA_WITH_RC4_128_MD5 RSA RSA RC4 128 MD5
0x00,0x05 TLS_RSA_WITH_RC4_128_SHA RSA RSA RC4 128 SHA
0x00,0x0A TLS_RSA_WITH_3DES_EDE_CBC_SHA RSA RSA 3DES EDE CBC SHA
0x00,0x2F TLS_RSA_WITH_AES_128_CBC_SHA RSA RSA AES 128 CBC SHA
0x00,0x35 TLS_RSA_WITH_AES_256_CBC_SHA RSA RSA AES 256 CBC SHA
0x00,0x3C TLS_RSA_WITH_AES_128_CBC_SHA256 RSA RSA AES 128 CBC SHA 256
0x00,0x3D TLS_RSA_WITH_AES_256_CBC_SHA256 RSA RSA AES 256 CBC SHA 256
0x00,0x9C TLS_RSA_WITH_AES_128_GCM_SHA256 RSA RSA AES 128 GCM SHA 256
0x00,0x9D TLS_RSA_WITH_AES_256_GCM_SHA384 RSA RSA AES 256 GCM SHA 384

Tools

These tools allow to list the cipher suites supported by a server.

Nmap

$ nmap --script ssl-enum-ciphers -p 443 www.exemple.com

TestSSLServer

$ java -jar TestSSLServer.jar www.exemple.com 443

CipherScan

$ git clone git://github.com/jvehent/cipherscan.git
$ cd cipherscan
$ ./cipherscan www.exemple.com
