How to create your PKI with EJBCA

Date mer. 06 juillet 2016 Modified mer. 06 juillet 2016 Tags Certificate / Debian

This post describe step by step how to install EJBCA on Linux Debian with Wildfly 10. Then, create your Certificate Authority to issue certificate.

Install Debian

  • Define the hostname : pki.intra.net

  • change time zone for UTC

# dpkg-reconfigure tzdata
Geographic area : none of the above
Time zone : UTC
  • Install NTP
# apt-get install ntp
# nano /etc/ntp.conf
server 0.fr.pool.ntp.org
server 1.fr.pool.ntp.org
server 2.fr.pool.ntp.org
server 3.fr.pool.ntp.org
  • Print date & time
# timedatectl

Install MariaDB

  • Install MariaDB server
# apt-get install mariadb-server
  • Start and enable MariaDB
# systemctl start mysql.service
# systemctl enable mysql.service
  • Create database
$ mysql -u root -p
MariaDB> CREATE DATABASE ejbca CHARACTER SET utf8 COLLATE utf8_general_ci;
MariaDB> GRANT ALL PRIVILEGES ON ejbca.* TO 'ejbca'@'localhost' IDENTIFIED BY 'ejbca';

Install Java

  • Download JDK from Oracle
# wget --header "Cookie: oraclelicense=accept-securebackup-cookie" http://download.oracle.com/otn-pub/java/jdk/8u92-b14/jdk-8u92-linux-x64.tar.gz
# tar -zxf jdk-8u92-linux-x64.tar.gz -C /opt/
# ln -s /opt/jdk1.8.0_92 /opt/jdk
  • Make a new JDK alternatives
# update-alternatives --install /usr/bin/java java /opt/jdk/bin/java 100
# update-alternatives --install /usr/bin/javac javac /opt/jdk/bin/javac 100
  • Update the default JDK
# update-alternatives --config java
# update-alternatives --config javac
  • Verification
# java -version
java version "1.8.0_92"
Java(TM) SE Runtime Environment (build 1.8.0_92-b14)
Java HotSpot(TM) 64-Bit Server VM (build 25.92-b14, mixed mode)
  • Install JCE (A priori inutile)
# wget --header "Cookie: oraclelicense=accept-securebackup-cookie" http://download.oracle.com/otn-pub/java/jce/8/jce_policy-8.zip
# unzip jce_policy-8.zip
# mv /opt/jdk/jre/lib/security/local_policy.jar /opt/jdk/jre/lib/security/local_policy.jar.backup
# mv /opt/jdk/jre/lib/security/US_export_policy.jar /opt/jdk/jre/lib/security/US_export_policy.jar.backup
# cp UnlimitedJCEPolicyJDK8/local_policy.jar /opt/jdk/jre/lib/security/
# cp UnlimitedJCEPolicyJDK8/US_export_policy.jar /opt/jdk/jre/lib/security/

Install Wildfly

  • Create a wildfly user
# groupadd -r wildfly
# useradd -r -g wildfly -d /opt/wildfly -s /sbin/nologin wildfly
  • Download Wildfly from http://wildfly.org/downloads/
$ wget http://download.jboss.org/wildfly/10.0.0.Final/wildfly-10.0.0.Final.zip
  • Decompress archive
# unzip wildfly-10.0.0.Final.zip -d /opt
# ln -s /opt/wildfly-10.0.0.Final /opt/wildfly
# chown -Rf wildfly:wildfly /opt/wildfly
# chown -Rf wildfly:wildfly /opt/wildfly-10.0.0.Final
  • Copy script for systemd
# mkdir /etc/wildfly
# cp /opt/wildfly/docs/contrib/scripts/systemd/wildfly.conf /etc/wildfly/
# cp /opt/wildfly/docs/contrib/scripts/systemd/wildfly.service /etc/systemd/system/
# cp /opt/wildfly/docs/contrib/scripts/systemd/launch.sh /opt/wildfly/bin/
# chmod +x /opt/wildfly/bin/launch.sh
  • Edit configuration file (if required)
# nano /opt/wildfly/standalone/configuration/standalone.xml
  • Start and enable Wildfly
# systemctl start wildfly.service
# systemctl enable wildfly.service
  • Add user
# /opt/wildfly/bin/add-user.sh
  • Browse to http://pki.intra.net:8080 (website) or http://pki.intra.net:9990 (admin console)

Configure WildFly for EJBCA (Part 1/2)

  • Download and install database driver for Java application
# wget https://downloads.mariadb.com/enterprise/80e4-qqsb/connectors/java/connector-java-1.4.6/mariadb-java-client-1.4.6.jar
# cp mariadb-java-client-1.4.6.jar /opt/wildfly/standalone/deployments/mariadb-java-client.jar
  • Increase allowed memory usage
# nano /opt/wildfly/bin/standalone.conf
JAVA_OPTS="-Xms256m -Xmx512m -XX:MetaspaceSize=256M -XX:MaxMetaspaceSize=256m -Djava.net.preferIPv4Stack=true"
  • Add datasource
# /opt/wildfly/bin/jboss-cli.sh -c
[standalone@localhost:9990 /] data-source add --name=ejbcads --driver-name="mariadb-java-client.jar" --connection-url="jdbc:mysql://127.0.0.1:3306/ejbca" --jndi-name="java:/EjbcaDS" --use-ccm=true --driver-class="org.mariadb.jdbc.Driver" --user-name="ejbca" --password="ejbca" --validate-on-match=true --background-validation=false --prepared-statements-cache-size=50 --share-prepared-statements=true --min-pool-size=5 --max-pool-size=150 --pool-prefill=true --transaction-isolation=TRANSACTION_READ_COMMITTED --check-valid-connection-sql="select 1;"
[standalone@localhost:9990 /] :reload
  • Configure WildFly Remoting
[standalone@localhost:9990 /] /subsystem=remoting/http-connector=http-remoting-connector:remove
[standalone@localhost:9990 /] /subsystem=remoting/http-connector=http-remoting-connector:add(connector-ref="remoting",security-realm="ApplicationRealm")
[standalone@localhost:9990 /] /socket-binding-group=standard-sockets/socket-binding=remoting:add(port="4447")
[standalone@localhost:9990 /] /subsystem=undertow/server=default-server/http-listener=remoting:add(socket-binding=remoting)
[standalone@localhost:9990 /] :reload
  • Configure logging

/!\ DEBUG enabled for this example.

[standalone@localhost:9990 /] /subsystem=logging/logger=org.ejbca:add
[standalone@localhost:9990 /] /subsystem=logging/logger=org.ejbca:write-attribute(name=level, value=DEBUG)
[standalone@localhost:9990 /] /subsystem=logging/logger=org.cesecore:add
[standalone@localhost:9990 /] /subsystem=logging/logger=org.cesecore:write-attribute(name=level, value=DEBUG)
  • Remove existing TLS and HTTP configuration
[standalone@localhost:9990 /] /subsystem=undertow/server=default-server/http-listener=default:remove
[standalone@localhost:9990 /] /socket-binding-group=standard-sockets/socket-binding=http:remove
[standalone@localhost:9990 /] :reload
  • Check running
[standalone@localhost:9990 /] :read-attribute(name=server-state)
{
    "outcome" => "success",
    "result" => "running"
}
  • Check datasource connection
[standalone@localhost:9990 /] /subsystem=datasources/data-source=ejbcads:test-connection-in-pool
{
    "outcome" => "success",
    "result" => [true]
}

Install EJBCA

  • install dependencies
# apt-get install ant ant-optional unzip
  • Download EJBCA (version 6.3.1.1)
# wget http://downloads.sourceforge.net/project/ejbca/ejbca6/ejbcare th_6_3_1_1/ejbca_ce_6_3_1_1.zip
# unzip ejbca_ce_6_3_1_1.zip -d /opt/
  • Edit configuration file
# cp /opt/ejbca_ce_6_3_1_1/conf/ejbca.properties.sample /opt/ejbca_ce_6_3_1_1/conf/ejbca.properties
# nano /opt/ejbca_ce_6_3_1_1/conf/ejbca.properties

appserver.home=/opt/wildfly

# cp /opt/ejbca_ce_6_3_1_1/conf/web.properties.sample /opt/ejbca_ce_6_3_1_1/conf/web.properties
# nano /opt/ejbca_ce_6_3_1_1/conf/web.properties

httpsserver.hostname=pki.intra.net

# cp /opt/ejbca_ce_6_3_1_1/conf/database.properties.sample /opt/ejbca_ce_6_3_1_1/conf/database.properties
# nano /opt/ejbca_ce_6_3_1_1/conf/database.properties

database.name=mysql
database.url=jdbc:mysql://127.0.0.1:3306/ejbca?characterEncoding=UTF-8
database.driver=org.mariadb.jdbc.Driver
database.username=ejbca
database.password=ejbca
  • Build and deploy EJBCA
# cd /opt/ejbca_ce_6_3_1_1/
# ant clean deployear
  • Install
# ant runinstall
  • Deploy TLS keystores to WildFly
# ant deploy-keystore

Configure WildFly for EJBCA (Part 2/2)

  • Configure TLS
# /opt/wildfly/bin/jboss-cli.sh -c
[standalone@localhost:9990 /] /interface=http:add(inet-address="0.0.0.0")
[standalone@localhost:9990 /] /interface=httpspub:add(inet-address="0.0.0.0")
[standalone@localhost:9990 /] /interface=httpspriv:add(inet-address="0.0.0.0")
[standalone@localhost:9990 /] /socket-binding-group=standard-sockets/socket-binding=http:add(port="8080",interface="http")
[standalone@localhost:9990 /] /subsystem=undertow/server=default-server/http-listener=http:add(socket-binding=http)
[standalone@localhost:9990 /] /subsystem=undertow/server=default-server/http-listener=http:write-attribute(name=redirect-socket, value="httpspriv")
[standalone@localhost:9990 /] :reload
[standalone@localhost:9990 /] :read-attribute(name=server-state)
  • Configure identities and socket bindings:
[standalone@localhost:9990 /] /core-service=management/security-realm=SSLRealm:add()
[standalone@localhost:9990 /] /core-service=management/security-realm=SSLRealm/server-identity=ssl:add(keystore-path="${jboss.server.config.dir}/keystore/keystore.jks", keystore-password="serverpwd", alias="prod-ica1")
[standalone@localhost:9990 /] /core-service=management/security-realm=SSLRealm/authentication=truststore:add(keystore-path="${jboss.server.config.dir}/keystore/truststore.jks", keystore-password="changeit")
[standalone@localhost:9990 /] /socket-binding-group=standard-sockets/socket-binding=httpspriv:add(port="8443",interface="httpspriv")
[standalone@localhost:9990 /] /socket-binding-group=standard-sockets/socket-binding=httpspub:add(port="8442", interface="httpspub")
[standalone@localhost:9990 /] :reload
[standalone@localhost:9990 /] :read-attribute(name=server-state)
  • Wait then continous
[standalone@localhost:9990 /] /subsystem=undertow/server=default-server/https-listener=httpspriv:add(socket-binding=httpspriv, security-realm="SSLRealm", verify-client=REQUIRED)
[standalone@localhost:9990 /] /subsystem=undertow/server=default-server/https-listener=httpspub:add(socket-binding=httpspub, security-realm="SSLRealm")
[standalone@localhost:9990 /] :reload
  • Finalize Wildfly configurations
[standalone@localhost:9990 /] /system-property=org.apache.tomcat.util.buf.UDecoder.ALLOW_ENCODED_SLASH:add(value=true)
[standalone@localhost:9990 /] /system-property=org.apache.catalina.connector.CoyoteAdapter.ALLOW_BACKSLASH:add(value=true)
[standalone@localhost:9990 /] /system-property=org.apache.catalina.connector.URI_ENCODING:add(value="UTF-8")
[standalone@localhost:9990 /] /system-property=org.apache.catalina.connector.USE_BODY_ENCODING_FOR_QUERY_STRING:add(value=true)
[standalone@localhost:9990 /] /subsystem=webservices:write-attribute(name=wsdl-host, value=jbossws.undefined.host)
[standalone@localhost:9990 /] /subsystem=webservices:write-attribute(name=modify-wsdl-address, value=true)
[standalone@localhost:9990 /] :reload

open https://pki.intra.net:8443/ejbca/adminweb/
