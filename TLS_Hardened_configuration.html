TLS: Hardened configuration

Date jeu. 23 juillet 2015 Modified dim. 02 août 2015 Tags Cryptography / Network

In recent years, many vulnerabilities (BEAST, CRIME, FREAK, ...) have been published about TLS protocol. Today, it is essential to enhanced configuration of TLS protocol to ensure the security of communication. This article lists the best practices to enhanced configuration of TLS protocol.

Protocol version

Version Recommended state Comment
1.0 Disabled Vulnerability: BEAST (Browser Exploit Against SSL/TLS)
1.1 Disabled Enable only for interoperability
1.2 Enable This is the last version

Compression method

Method Recommended state Comment
Null Enable  
Deflate Disabled Vulnerability: CRIME (Compression Ratio Info-leak Made Easy)

Cryptographic algorithm

Key exchange

Algorithm Key length Recommended state Comment
RSA Export 512 bits Disabled Vulnerability: FREAK (Factoring RSA Export Keys)
RSA 2048 bits Disabled Enable only for interoperability
DH 2048 bits Disabled Enable only for interoperability
DHE 2048 bits Enable Allow PFS (Perfect Forward Secrecy)
ECDHE 256 bits Enable Allow PFS (Perfect Forward Secrecy)

Encryption

Algorithm Key length Recommended state Comment
DES 56 bits Disabled Weak algorithm
3DES 112 bits Disabled Weak algorithm
RC4 128 bits Disabled Weak algorithm
AES-CBC 128 bits Disabled Vulnerability: Lucky 13
AES-CBC 256 bits Disabled Vulnerability: Lucky 13
AES-GCM 128 bits Enable Allow AEAD (Authenticated Encryption with Associated Data)
AES-GCM 256 bits Enable Allow AEAD (Authenticated Encryption with Associated Data)

Message Authentication Code

Algorithm Key length Recommended state Comment
MD5 128 bits Disabled Weak algorithm
SHA-1 160 bits Disabled Weak algorithm
SHA-2 256 bits Enable
